1. Overview

This document details the architectural blueprints of the MCL Internship Management Portal. The system employs a 3-tier architecture: the Client Presentation layer, the server-side Routing & Logic execution layer, and the database persistence layer.

🛡️ Security Standard: The implementation strictly separates public database queries (governed by Row-Level Security) from system-level API transactions (governed by the Supabase Service Role configuration).

2. Master Architecture Diagram

The master architecture details data ingestion and access rules. Client devices run SPA operations, querying public scopes using Row-Level Security (RLS) filters. Server-side API endpoints handle mail dispatches, PDF builds, and service role overrides securely.

3. Certificate Generation & Verification Sequence

To ensure verification authenticity, the issuance process generates an immutable UUIDv4 registry entry. The QR code links directly to this record on the live server, exposing document tampering.
