3. Certificate Generation & Verification Sequence

This sequence details the verification structure of the completion certificates. When issued, the Next.js API server fetches the required metadata and saved digital signature coordinates from PostgreSQL, draws them into the PDF, and adds a database-matched validation QR code.

sequenceDiagram autonumber actor Admin as Area Admin participant Server as Next.js API Server participant DB as PostgreSQL Database participant Storage as Object Storage actor Student as Student / Employer Note over Admin, Student: Phase A: Document Generation Admin->>Server: Click "Issue Certificate" (on Pending tab) Server->>DB: Query Student info, Mentor signature, & Admin signature DB-->>Server: Return metadata & Base64 signature vectors Server->>Server: Generate unique UUIDv4 & Validation URL Server->>Server: Generate QR Code (linking to verify URL) Server->>Server: Compile PDF (using pdf-lib) and draw QR + Signatures Server->>Storage: Upload generated PDF buffer Server->>DB: Update internship record with signed URL Server->>Student: Email PDF as attachment Note over Admin, Student: Phase B: Real-Time Scanner Verification Student->>Student: Scan QR Code on Certificate Student->>Server: Request /verify/{uuid} Server->>DB: Query official registry by UUID DB-->>Server: Return authenticated record Server-->>Student: Display verified Student Name, Dates, and Project Title
