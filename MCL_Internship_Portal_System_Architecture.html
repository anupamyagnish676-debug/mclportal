1. Entire System Architecture

The MCL Internship Management Portal uses a highly optimized 3-tier architecture. Client devices run Single Page Application (SPA) dashboards, making standard authenticated requests directly to the database. These queries are intercepted and filtered by database **Row-Level Security (RLS)**. Complex backend transactions (like user onboarding, emailing credentials, and compiling certificate PDFs) run inside secure Next.js API server routes.

2. LOR Submission & Student Onboarding Flow

The LOR flow facilitates manual student registration by bridging employee submissions with direct Area Admin controls. This workflow ensures that student records are validated by central screening before they are registered in the system.

📬 Onboarding Detail: Disabling initial screening emails ensures that the student receives exactly one combined welcome letter containing LOR approval confirmation, internship period dates, and portal credentials.

3. Certificate Issuance & Verification Lifecycle

The certificate pipeline secures the completion documents from forgery. The generated PDF incorporates digital signatures and a database-matched verification QR code.
